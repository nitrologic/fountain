class Pixmap16 {
    pix4: Uint32Array;    
    width: number;
    height: number;
    span: number;

    constructor(w: number, h: number) {
        this.width = w;
        this.height = h;
	this.span = (w+7)>>2;
        this.pix4 = new Uint32Array(this.span * h);
    }

    plot(x: number, y: number, c4: number): void {
	let shift=4*(7-(x&7));
	let pos=(x>>3)+y*this.span;
	this.pix4[pos]|=c4<<shift;
    }

    rect(x: number, y: number, w: number, h: number, c4: number): void {
	let c32 = c4 * 0x11111111;
	let x0 = x >> 3;
	let x1 = (x + w - 1) >> 3;
	let leftMask = ~0 << (4 * (8 - (x & 7)));
	let rightMask = ~0 >>> (4 * ((x + w) & 7));
	
	for (let row = y; row < y + h; row++) {
		let pos = row * this.span + x0;
		
		if (x0 === x1) {
		// Single word
		this.pix4[pos] |= c32 & (leftMask & rightMask);
		} else {
		// Left edge
		this.pix4[pos++] |= c32 & leftMask;
		
		// Middle full words
		for (let i = x0 + 1; i < x1; i++) {
			this.pix4[pos++] = c32;
		}
		
		// Right edge
		this.pix4[pos] |= c32 & rightMask;
		}
	}
    }

    char(cx: number, cy: number, c:number, c4: number): void {
        for (let y = 0; y < 8; y++) {
            let bits = font[c * 2 + (y >> 2)];
            let bit = 1 << (31 - 8 * (y & 3));
            for (let x = 0; x < 8; x++) {
                if (bits & bit) this.plot(cx + x, cy + y, c4);
                bit >>= 1;
            }
        }
    }
}

structure. However, there are two key issues in `prepareGeminiPrompt` and one potential crash in
`connectGoogle` related to your recent tool calling and continuity fixes.

1. Continuity Issue in `prepareGeminiPrompt`
In your `assistant` case, you are checking for `item.toolcallid` instead of
`item.toolcalls`. When the assistant makes a tool call, OpenAI format uses `toolcalls`.
Because of this typo, the code falls through to the `else` block and pushes an empty text part
instead of the required `functionCall` object, breaking Gemini's conversation history.

Additionally, in the `tool` case, Gemini expects the role to be `"function"`, not `"tool"`.

Here is the corrected switch block for `prepareGeminiPrompt`:


			case "assistant": {
				if (item.tool_calls) {
					const parts = item.tool_calls.map(tc => ({
						functionCall: {
							name: tc.function.name,
							args: JSON.parse(tc.function.arguments)
						}
					}));
					contents.push({ role: "model", parts });
				} else {
					contents.push({ role: "model", parts: [{ text: item.content || "" }] });
				}
				break;
			}
			case "tool": {
				const functionResponse = { name: item.name, response: JSON.parse(text) };
				if (debugging) echo("[GEMINI] functionResponse", functionResponse);

2. Potential Crash in `connectGoogle`
When Gemini decides to call a function, it often omits text entirely. Calling `await
result.response.text()` will throw an error if no text parts exist in the response, terminating the
relay.

You should wrap the text extraction in a try-catch block:


						let text = "";
						try {
							text = await result.response.text();
						} catch (e) {
							// Ignored: model returned a pure tool call without text
						}

						const usage = result.response.usageMetadata || {};

						const calls = result.response.functionCalls();

in the history, the tool results will be returned as `functionResponse` parts, and the strict
alternating continuity required by the Gemini API will be maintained.

### Looking at your current setup, you have a linear pipeline: **User → Fountain → Sloppy → Discord**. To make this a true social hub with multiple models and users, you need to move from a relay architecture to a **message bus with routing logic**.

Here are architectural options leveraging your existing `slopBroadcast`, `rohaHistory`, and the `forge` session system:

## 1. Multi-Model Orchestration Modes

Instead of `relay(0)` calling one model, implement **conversation topologies**:

**Round-Robin Panel** (`/panel mode`)
- User message broadcasts to N models sequentially
- Each model sees the previous model's response in history
- Sloppy posts threaded replies: "Claude says…", "GPT says…"
- Users vote with reactions to select the "winner" that continues

**Debate Mode** (`/debate`)
- Two models argue opposing sides
- `rohaHistory` branches using your existing `sessionStack`/`pushHistory`
- Each model gets isolated context of just their side + the debate topic
- Sloppy relays both streams with "Pro:" and "Con:" headers

**Ensemble Mode** (`/ensemble`)
- Parallel async relays (your `beginRelay()` is already async-capable)
- Wait for all models, then use a "synthesizer" prompt to merge responses
- Post the synthesis as the official reply, with model attributions in a spoiler tag
